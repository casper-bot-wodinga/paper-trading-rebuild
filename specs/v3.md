## Bankroll System — Self-Calibrating Risk Ceiling

The per-tick risk ceiling self-adjusts based on Kairos's actual performance. Starts small, grows with skill.

**Critical: the bankroll is a ceiling, not a target.** The agent doesn't have to use the whole budget every tick. It's the max allowed risk — Kairos can stay under it whenever conviction is low.

### Mechanics
- **Starting ceiling per tick**: $35 (floor, never goes below)
- **Win a closed trade** → ceiling × 1.01 (increase by 1%)
- **Lose a closed trade** → ceiling × 0.99 (decrease by 1%)
- **Adjustable param** — the 1% growth/decay rate can be tuned
- **No cap** (but natural ceiling emerges from win rate)
- The ceiling can be split any way across any number of stocks per tick:
  - 5 × $7 probes on random tickers (early exploration)
  - 1 × $35 conviction play (when you're sure)
  - 3 × $11.67 on sector themes
  - 0 × $0 if nothing looks good (patience is valid)

### Why Percentage Works Better

Linear +/-$1 is clunky early and trivial late:
- At $35, +$1 = 2.8% boost (fine but arbitrary)
- At $350, +$1 = 0.28% boost (negligible)

Percentage scales naturally with confidence:
- At $35, +1% = +$0.35 (small, proportional)
- At $350, +1% = +$3.50 (meaningful at scale)
- The growth rate is consistent regardless of bankroll size

### Why This Works

| Phase | Ceiling | Behavior |
|-------|---------|----------|
| Early exploration | $35-$50 | Wide net, lots of small probes across many tickers. More data = better replay. |
| Learning | $50-$80 | Win rate pulling away from losses. Trades bigger but more selective. Fewer tickers. |
| Competent | $80-$150 | Clear pattern recognition. Fewer trades, larger amounts. Conviction-driven. |
| Confident | $150+ | Betting on high-conviction plays. Infrequent but significant. |

The agent doesn't have a risk model — it **is** the risk model. Every closed trade is a training signal that adjusts future sizing.

### Replay Integration
Night replay can simulate:
- "What if I used this bankroll system for the past N days?"
- "What if my risk ceiling was 2x/0.5x on each trade?"
- "What if I sized up on winners and down on losers more aggressively?"
- "What if I'd used a flat $50 instead of the bankroll system?"

Each permutation becomes a branch the agent scores and journals about. The bankroll doesn't just track performance — it's a dial the agent learns to tune.

## Off-Market Replay (night mode)

After market close, each trader enters replay mode. Instead of Python virtual runners:

1. **Cron fires** — "Night replay — run on last 7 days from DB"
2. **Read current strategies** + **load replay context** from Postgres (quotes, signals, portfolio states)
3. **Read bankroll** — current ceiling from `bankroll.md`
4. **Replay each tick** — same decision loop as live, but against past data
5. **"What if" branching** — on any replay tick, fork and ask:
   - "what if I'd picked $STOCK instead?"
   - "what if my risk size was 2x/0.5x?"
   - "what if I'd sat out this trade entirely?"
6. **Score** — compare branch outcomes against actual results, adjust simulated bankroll
7. **Pattern recognition** — "Buying tech on Monday after a Friday dip wins 70% of the time"
8. **Update strategies/active.md** — add what worked, demote what didn't
9. **Journal the replay** — "Tried NVDA vs AMD over last week, NVDA gave 2.3% more return"

Night replay replaces the old `virtual_runner.py` + `promote_virtual_to_live.py` pattern entirely.

## How It Learns & Improves

### Layer 1 — Per Tick (immediate)
Agent reads `data-bus__get_self_stats()` every tick. Win rate by signal, sector performance, confidence calibration. Adjusts conviction thresholds in real-time. No ML needed — statistical self-awareness.

### Layer 2 — EOD Reflection (per day)
Agent reviews today's journal entries, identifies patterns, updates MEMORY.md. Reads yesterday's + today's journal on each session start for continuity.

### Layer 3 — Night Replay (ML + what-if)
Replays from Past N days from Postgres. Tries alternative picks at branching points. Also tries different risk sizes — "what if I'd gone 2x on that winning trade?" or "what if I'd halved my position?" Scores outcomes. Updates `strategies/active.md` with what worked.

### Layer 3.5 — Adaptive Strategy Evolution
Every X closed trades, the agent distills journal patterns into `strategies/active.md`:

```
X = BASE_X × (1 - win_rate_ema) + 1
```

- **BASE_X** = configurable starting cadence (e.g., 10 trades)
- **win_rate_ema** = exponential moving average of recent win rate (0.0 to 1.0)
- **High win rate** (0.7) → X grows (~4x) → slower strategy updates → lock in what works
- **Low win rate** (0.3) → X shrinks (~3x) → faster strategy updates → try new things
- **Floor**: X never below 3 (always getting some reps before changing)

This means:
- A hot streak locks in the winning playbook. You don't change what's working.
- A cold streak loosens the system. The agent tries new approaches faster.
- The strategy file stays lean — just the current winning playbook, not a dump.

The cadence itself is a meta-parameter the replay engine can optimize.

### Layer 4 — ML Signal Optimization
The learning loop runs as a background script against Postgres. Optimization sweeps write params to `params.json` in the agent's workspace. The agent reads it on next tick.

## What This Unlocks

1. **Strategic thinking** — Per-position thesis files let the agent hold multi-week perspectives: "I'm down 2.5% but my catalyst is 4 weeks out, thesis intact"
2. **Self-improving playbook** — `strategies/active.md` evolves from every journal + replay cycle. Strategies get promoted, demoted, and refined.
3. **"What if" power** — Night replay lets the agent try different picks AND different risk sizes against history. "NVDA worked better than AMD in this regime" AND "2x sizing on those wins would've been +$40 instead of +$20"
4. **Character depth** — Journaling as a character builds narrative consistency and makes the agent reflect as its persona would
5. **Adaptive learning cadence** — The strategy update rate self-calibrates. Winning → lock in. Losing → explore. The agent learns how fast to learn.
6. **Clean split** — Postgres = structured replay data. Workspace = agent's living memory. No overlap.
7. **Natural iteration** — Change strategies/active.md, next tick picks it up. No rebuild, no migration.

---

**TL;DR**: You already built the infrastructure. The simplification is making the agents **be** OpenClaw agents. Everything else — data bus, learning loop, executor — becomes tooling the agent uses, not infrastructure the agent lives inside.
